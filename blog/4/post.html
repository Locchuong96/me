Intro

Carbon, or Carbon-Lang, is an experimental, general-purpose programming language. The project is open-source and was started by Google, following in the footsteps of previous Google-made programming languages (Go and Dart). Google engineer Chandler Carruth first introduced Carbon at the CppNorth conference in Toronto in July 2022. He stated that Carbon was created to be a C++ successor. The language is expected for a 1.0 release to occur in 2024 or 2025. They say:
- JavaScript → TypeScript
- Java → Kotlin
- C++ → Carbon

As the development team annouced, Carbon will have these adavances:

Fast and works with C++

With LLVM (a set of compiler and toolchain technologies that can be used to develop a front end for any programming language and a back end for any instruction set architecture.) Carbon's performance will be match C++, and this language can access to bits and addresses (also know as the memory addresses).
Carbon can work with your exisiting C++ code. from inheritance to templates. So we don't worry about build everything from scratch. So carbon is a compiled language.

Modern and evolving

The syntax quite similar to C++, and an easy,tool-based upgrades between each version of Carbon.

Welcoming open-source community

As Python, the development team define Carbon as a open language, and they wellcome the other contributors.This is really a bright point of view. You can see the successful of Python when they build an community.They said Batteries-included approach: compiler, libraries, docs, tools, package manager, and more...

Why they build Carbon Programming language?

When we need an performance-critical software, The answer is C++. Today C++ is not an popular language as 20th century. But you know the ranking is not everything. There are the differences between usages of them. In this century ,the hardware systems become clearly and easy for software developer. In this century, we are building the applications, website, APIs services,etc, and we build them with packages, tools, libraries...That is the reason why people does not use C++ as 20th century.
But improving C++ is extremly difficult because alot of problems accumulated from time to time. The developement team have an description about it here They call these problems is technical debts :). So their solution is to void inheriting the legacy of C or C++ directy, and instead start with solid language foundations like modern generics system, modular code organization, and consistent, simple syntax. Alot of existing modern langugae already provide an excellent developer experience as Go,Swift,Rust and many more . But these languages present significant barries to adoption and migration from C++. And it seem like C++ legacy is so big to avoid.
Carbon is the solution. It is designed around interoperability with C++ as well as large-scale adoption and migration for existing C++ codebases and developers.

Language goals

You can read the detail from the development team here. The summary of their goals
- Performance-critical software
- Software and language evolution
- Code that is easy to read, understand, and write
- Practical safety and testing mechanisms
- Fast and scalable development
- Modern OS platforms, hardware architectures, and environments
- Interoperability with and migration from existing C++ code
And they clearly say that, a stable application interface (ABI) and perfect backwards or forwards compability is not in this goal. Meaning we still have alot of work in the future for next candidates.
